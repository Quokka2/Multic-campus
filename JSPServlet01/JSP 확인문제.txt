1. 웹 프로그램의 3단계
클라이언트, 웹서버, DB
2. 웹서버측의 구성 : 웹서버 + WAS, 2개 각각의 역할
웹서버(http요청/응답), WAS(자바로 동적처리부분 담당)
3. WAS는 무엇의 약자?
Web Application Server
4. 웹서버 종류?
apache, nginx(엔진엑스), IIS
5. 어떤 자바 WAS를 써봤나요?
tomcat
6. JDBC 4단계를 정리해보세요.
1) 드라이버 설정 2) DB연결 3) SQL문 생성 4) DB로 SQL문 전송
7. mysql과 oracle의 기본 port는?
3306/1521
8. JSP는 무엇의 약자?
Java Server Pages
9. servlet과 JSP의 관계?
JSP는 서블릿이라 자바프로그램으로 변환되어 실행
10. http의 용도, 무엇의 약자?
클라이언트와 웹서버의 통신 규칙(규약, 프로토콜), hyperText transfer protocol
11. http의 구성요소 2가지
header + body
12. 브라우저가 보낼 때 http 헤더와 바디
header(요청주소 + get(데이터)) / body(post(데이터))
13. 웹서버가 보낼 때 http헤더와 바디
header(상태코드) / body(처리결과)
