1. 웹 프로그램의 3단계
웹 프로그램의 3단계는 클라이언트 측에서 실행되는 HTML, CSS, JavaScript 등의 정적 페이지
서버 측에서 실행되는 동적 페이지를 생성하는 어플리케이션 서버, 그리고 데이터베이스 서버로 나뉩니다.
2. 웹서버측의 구성 : 웹서버 + WAS, 2개 각각의 역할
웹서버는 정적인 웹 페이지를 제공하고, WAS는 동적인 웹 페이지를 생성하는 역할을 합니다.
웹서버는 클라이언트와의 연결을 담당하고
WAS는 웹 애플리케이션의 비즈니스 로직 처리 및 데이터베이스와의 연동 등을 담당합니다.
3. WAS는 무엇의 약자?
WAS는 Web Application Server의 약자입니다.
4. 웹서버 종류?
웹서버의 종류로는 Apache, Nginx, Microsoft IIS, Lighttpd 등이 있습니다.
5. 어떤 자바 WAS를 써봤나요?
저는 Tomcat, JBoss, WebLogic, WebSphere 등의 자바 WAS를 사용해봤습니다.
6. JDBC 4단계를 정리해보세요.
단계 1: 드라이버 로딩
단계 2: 데이터베이스 연결
단계 3: SQL문 실행
단계 4: 결과 처리 및 연결 종료
7. mysql과 oracle의 기본 port는?
mysql의 기본 port는 3306이고, oracle의 기본 port는 1521입니다.
8. JSP는 무엇의 약자?
JSP는 Java Server Pages의 약자입니다.
9. servlet과 JSP의 관계?
Servlet과 JSP는 모두 웹 애플리케이션을 개발하기 위한 자바 기술입니다.
Servlet은 자바 코드로 작성된 웹 애플리케이션 로직을 처리하고, JSP는 HTML 코드 안에 자바 코드를 넣어서 동적인 웹 페이지를 생성합니다.
Servlet과 JSP는 서로 보완적인 역할을 하며, 둘을 함께 사용하여 웹 애플리케이션을 개발합니다.
10. http의 용도, 무엇의 약자?
HTTP는 HyperText Transfer Protocol의 약자로, 웹 브라우저와 웹 서버 간에 데이터를 주고받기 위한 통신 규약입니다.
HTTP는 클라이언트에서 요청(request)을 보내고, 서버에서 응답(response)을 반환하는 방식으로 동작합니다.
11. http의 구성요소 2가지
HTTP의 구성요소로는 요청(request)과 응답(response)이 있습니다.
요청은 클라이언트가 서버에 보내는 데이터로, HTTP 메서드(GET, POST, PUT, DELETE 등), URL, 헤더, 바디 등으로 구성됩니다.
응답은 서버가 클라이언트에 보내는 데이터로, 상태 코드, 헤더, 바디 등으로 구성됩니다.
12. 브라우저가 보낼 때 http 헤더와 바디
브라우저가 보낼 때 HTTP 헤더는 요청 메시지에 대한 부가 정보를 담고 있습니다.
예를 들어, 브라우저의 종류나 버전, 요청한 URL, 요청 메서드, 쿠키, 인증 정보 등이 HTTP 헤더에 포함됩니다.
HTTP 바디는 요청 메시지의 본문 데이터를 담고 있습니다.
예를 들어, POST 메서드를 사용할 때는 HTTP 바디에 사용자가 입력한 데이터가 포함됩니다.
13. 웹서버가 보낼 때 http헤더와 바디
웹서버가 보낼 때 HTTP 헤더는 응답 메시지에 대한 부가 정보를 담고 있습니다.
예를 들어, 응답 코드(200 OK, 404 Not Found 등), 응답한 데이터의 종류, 쿠키, 캐시 제어 등이 HTTP 헤더에 포함됩니다.
HTTP 바디는 응답 메시지의 본문 데이터를 담고 있습니다.
예를 들어, HTML 페이지나 이미지 파일 등이 HTTP 바디에 포함됩니다.
